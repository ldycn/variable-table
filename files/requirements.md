请尝试实现一个简化版的变量表编辑器，支持变量的增、删、改、查功能。

核心功能
表格管理：显示变量列表，包括名称、数据类型、默认值和注释。

数据操作：支持添加行、删除行以及编辑单元格内容。

数据校验：校验名称的唯一性、数据类型以及默认值格式。

支持两种数据类型：BOOL 和 INT（32 位有符号整数）。

可以使用任意 CSS 框架和组件库

代码结构清晰，组件职责明确

请遵循以下验收标准：

AC 1：表格展示
场景：打开变量表编辑器页面

当：页面加载完成

那么：

显示一个空表格，包含以下列：索引、名称、数据类型、默认值、注释

索引列为只读且自动生成

显示“添加行”和“删除行”按钮

===== 第 3 页 =====

AC 2：添加变量行
场景：表格为空或有数据

当：用户点击“添加行”按钮

那么：

在表格末尾添加一行新行

新行中所有字段（名称、数据类型、默认值、注释）默认为空

索引自动生成为当前最大索引 + 1

AC 3：删除变量行
场景：表格至少有一行数据

当：用户选中一行并点击“删除行”按钮

那么：

该行从表格中删除

后续行的索引自动重新计算

AC 4：编辑变量名称
场景：表格中有数据行

当：用户点击名称单元格并输入新名称

那么：

如果输入为空，则恢复原值并显示错误提示

如果输入的名称在表中已存在（不区分大小写），则显示错误“名称已存在”且不允许保存

如果输入的名称唯一且不为空，则保存成功

示例：

表中已有 counter

用户输入 Counter → 显示错误提示

用户输入 newCounter → 保存成功

===== 第 4 页 =====

AC 5：选择数据类型
场景：当前类型为 BOOL，默认值为 TRUE

当：用户双击数据类型单元格

那么：

显示下拉框，包含两个选项：BOOL 和 INT

选择后，单元格显示选中的类型

并且当：用户切换数据类型

那么：

默认值单元格自动更新为新类型的默认值（BOOL → TRUE，INT → 0）

示例：

当前类型为 BOOL，默认值为 TRUE

用户将类型改为 INT → 默认值自动变为 0

AC 6：编辑 BOOL 类型默认值
场景：某变量行的数据类型为 BOOL

当：用户点击默认值单元格并输入一个值

那么：

接受以下输入：true, false, TRUE, FALSE（不区分大小写）

显示格式统一为大写：TRUE 或 FALSE

如果输入其他值，则显示错误提示

示例：

用户输入 TRUE → 显示 TRUE

用户输入 false → 显示 FALSE

用户输入 yes → 显示错误提示

AC 7：编辑 INT 类型默认值
场景：某变量行的数据类型为 INT

当：用户点击默认值单元格并输入一个值

===== 第 5 页 =====

那么：

接受整数输入，范围为 -2,147,483,648 到 2,147,483,647

如果输入不是整数或超出范围，则显示错误提示

示例：

用户输入 42 → 保存成功

用户输入 3.14 → 显示错误提示

用户输入 9999999999 → 显示错误提示（超出范围）

AC 8：编辑注释
场景：表格中有数据行

当：用户点击注释单元格并输入文本

那么：

接受任意文本输入

值可以为空
